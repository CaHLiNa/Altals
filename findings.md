# Findings

## 2026-03-19

- 用户给出的待修问题只有 4 项，来源文件位于桌面，内容明确，不需要额外需求澄清。
- 当前仓库已经存在一批未提交修改，主要集中在 AI/编辑器工具栏相关文件，后续修复需要避免覆盖这些改动。
- `自动保存 / 关闭前保存确认` 当前只有 CodeMirror 侧的自动保存监听，文件级脏状态与关闭拦截链路基本未接通；`dirtyFiles` 只在 UI 上预留了标记位。
- `文件版本历史` 的 UI、提交对话框、git log/show 基本都已存在，但仓库没有 `git init` 能力，非 git 工作区下历史几乎不可用。
- `@ 文件搜索` 的底层文件索引存在，但弹层只做了简单 `includes` 过滤并硬截断 20 条，没有评分排序，也没有对当前工作区上下文做任何提升。
- `LaTeX 中文路径` 还需要继续下钻 Rust 编译链路；目前已确认问题不在前端 compile 调用入口，而更可能在后端编译命令与路径处理细节。
- 版本历史后续回归排查显示，问题不只是“没做 `git init`”。在部分已有 `.git` 的工作区中，Rust `git2` 路径处理会让 `git_log` 失败后被前端吞成空数组，从而表现成“没有历史”；首次种子提交链路里还会进一步触发 `invalid path`。
- 本地版本历史场景下，系统 `git` CLI 在同一工作区可正常返回日志和文件内容，因此更稳妥的修复策略是保留远程相关能力在 `git2`，但把本地历史依赖的基础命令切到 CLI。
